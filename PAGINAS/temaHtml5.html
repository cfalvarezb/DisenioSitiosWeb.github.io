HTML5

¿Qué hay de nuevo en HTML5?

La declaración DOCTYPE para HTML5 es muy simple: <!DOCTYPE html>.

La declaración de codificación de caracteres (charset) también es muy simple: <meta charset="UTF-8">.

La codificación de caracteres predeterminada en HTML5 es UTF-8.


Nuevos elementos HTML5

Los nuevos elementos HTML5 más interesantes son:

Nuevos elementos semánticos como <header>, <footer>, <article>, y <section>.

Nuevos atributos de elementos de formulario como número, fecha, hora, calendario y rango.

Nuevos elementos gráficos : <svg> y <canvas>.

Nuevos elementos multimedia : <audio> y <video>.

Nuevas API HTML5 (interfaces de programación de aplicaciones)


nuevas API más interesantes en HTML5 son:

  • Geolocalización HTML
  • Arrastrar y soltar HTML
  • Almacenamiento local HTML
  • Caché de aplicaciones HTML
  • Trabajadores web HTML
  • HTML SSE

Tip: El almacenamiento local HTML es un poderoso reemplazo para las cookies.

Historia HTML

Desde los primeros días de la World Wide Web, ha habido muchas versiones de HTML:

Año Version
1989 Tim Berners-Lee invento www
1991 Tim Berners-Lee invento HTML
1993 Dave Raggett redacto HTML+
1995 HTML Working Group definio HTML 2.0
1997 Recomendacion del W3C HTML 3.2
1999 Recomendacion del W3C HTML 4.01
2000 Recomendacion del W3C XHTML 1.0
2008 WHATWG HTML5 Primer borrador público
2012 WHATWG HTML5 Living Standard
2014 Recomendacion W3C: HTML5
2016 Recomendacion de candidatos para W3C: HTML 5.1
2017 Recomendación W3C: HTML5.1 2a edición

De 1991 a 1999, HTML se desarrolló de la versión 1 a la versión 4.

En el año 2000, el Consorcio World Wide Web (W3C) recomendó XHTML 1.0. La sintaxis XHTML era estricta, y los desarrolladores se vieron obligados a escribir código válido y "bien formado".

En 2004, W3C decidió cerrar el desarrollo de HTML, a favor de XHTML.

En 2004, se formó WHATWG (Grupo de trabajo de tecnología de aplicación de hipertexto web). El WHATWG quería desarrollar HTML, de acuerdo con la forma en que se usaba la web, a la vez que era compatible con versiones anteriores de HTML.

En 2004 - 2006, el WHATWG obtuvo el apoyo de los principales proveedores de navegadores.

En 2006, el W3C anunció que apoyarían a WHATWG.

En 2008, se lanzó el primer borrador público HTML5.

En 2012, WHATWG y W3C decidieron una separación:

WHATWG quería desarrollar HTML como un "estándar de vida" . Un nivel de vida siempre se actualiza y mejora. Se pueden agregar nuevas funciones, pero no se pueden eliminar las funcionalidades antiguas.

El WHATWG HTML5 Living Standard se publicó en 2012 y se actualiza continuamente.

W3C quería desarrollar un estándar HTML5 y XHTML definitivo.

La Recomendación W3C HTML5 se lanzó el 28 de octubre de 2014.

La Recomendación W3C HTML5.1 2nd Edition se lanzó el 3 de octubre de 2017.

La Recomendación W3C HTML5.2 se lanzó el 14 de diciembre de 2017.


Soporte de navegador HTML5

HTML5 es compatible con todos los navegadores modernos.

Además, todos los navegadores, antiguos y nuevos, manejan automáticamente elementos no reconocidos como elementos en línea.

Debido a esto, puede "enseñar" a los navegadores antiguos a manejar elementos HTML "desconocidos".


Definir elementos semánticos como elementos de bloque

HTML5 define ocho nuevos elementos semánticos . Todos estos son elementos a nivel de bloque .

Para asegurar el comportamiento correcto en navegadores antiguos, puede configurar la propiedad de visualización CSS para que bloqueen estos elementos HTML :

header, section, footer, aside, nav, main, article, figure {display: block;}
